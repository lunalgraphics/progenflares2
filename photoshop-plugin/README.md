# Progen Flares 2 — Photoshop Plugin

Adobe Photoshop UXP plugin that embeds the Progen Flares 2 editor in a modal WebView and exports the rendered flare as a screen-blended smart object layer.

## Requirements

- Adobe Photoshop 23.3.0 or later (UXP 6.0+)
- A document must be open before launching the plugin

## How It Works

1. The plugin panel shows two buttons: **New Lens Flare** and **Edit Selected**
2. Clicking "New Lens Flare" opens a modal dialog with the flare editor WebView
3. The current document composite is sent to the editor as a reference image
4. When the user clicks "Finish", the raw pixel data is sent back to Photoshop
5. The plugin creates a smart object containing the render layer + a hidden metadata text layer
6. The smart object is set to Screen blend mode

### Editing Existing Flares

Selecting a previously created smart object and clicking "Edit Selected" re-opens the editor with the stored settings, allowing non-destructive re-editing.

## Building

From the repository root:

```bash
npm run build:photoshop
```

This outputs the WebView bundle to `photoshop-plugin/webview-contents/build/`.

## Loading in Photoshop

1. Open Photoshop
2. Go to **Plugins → Development → Load Plugin**
3. Select the `photoshop-plugin/` folder (or the `manifest.json` file)
4. The Progen Flares 2 panel will appear in the plugins sidebar

## File Structure

```
photoshop-plugin/
├── manifest.json           UXP plugin manifest (v5)
├── index.html              Plugin panel HTML (launch buttons)
├── index.js                Plugin logic (layer creation, messaging, modal)
├── icons/                  Plugin panel icons
└── webview-contents/       Built Svelte app (generated by build:photoshop)
    ├── index.html
    └── build/
        ├── bundle.js
        └── bundle.css
```

## Notes

- The `__IS_PHOTOSHOP_PLUGIN__` flag is injected at build time via Rollup's replace plugin
- Pixel data is transferred as base64-encoded RGBA between the WebView and UXP host
- User presets are persisted via `window.localStorage` within the UXP context
