# Progen Flares 2 — Photopea Plugin

Web-based plugin for [Photopea](https://www.photopea.com) that opens the Progen Flares 2 editor in a popup window and places the rendered flare as a screen-blended layer.

## How It Works

1. Photopea loads the plugin in an iframe
2. The plugin exports the current document as a PNG to get its dimensions
3. A popup window opens with the full flare editor UI
4. The current document is sent as a reference image to the popup
5. When the user clicks "Finish", the final flare image is sent back
6. The plugin uses the Photopea scripting API to create a new screen-blended layer

## Building

From the repository root:

```bash
npm run build:photopea
```

This outputs the editor bundle to `photopea-plugin/frame-contents/build/`.

## Testing Locally

1. Build with `npm run build:photopea`
2. Serve the `photopea-plugin/` directory on a local server (e.g. port 1995)
3. Open Photopea with the plugin URL encoded in the hash:
   ```
   https://www.photopea.com/#{"environment":{"plugins":[{"name":"pgf2","url":"http://localhost:1995/","icon":"http://localhost:1995/icon120.png"}]}}
   ```

## File Structure

```
photopea-plugin/
├── index.html              Plugin iframe page (loading spinner + launch logic)
├── code.js                 Plugin entry point (popup creation, Photopea API calls)
├── icon120.png             Plugin icon for Photopea's panel
└── frame-contents/         Built Svelte app (generated by build:photopea)
    ├── index.html
    └── build/
        ├── bundle.js
        └── bundle.css
```

## Notes

- The `__IS_PHOTOPEA_PLUGIN__` flag is injected at build time
- Communication between the popup and plugin iframe uses `window.postMessage`
- The popup must be allowed by the browser (popup blockers will prevent it)
- Uses the [Photopea API](https://www.photopea.com/api/) for scripting layer operations
